Paradigmas de Computação

Paradigmas são os modelos os quais as ferramentas(linguagens) serão utilizadas para resolver

Imperativo

     Esse paradigma segue o conceito de um estado e de ações que manipulam esse estado, nele encontramos procedimentos que servem de mecanismos de estruturação.      Podemos denominá-lo de procedural por incluir sub-rotinas ou procedimentos para estruturação. Ex:      -> Ada      -> ALGOL      -> C      -> Basic

Funcional

     Este paradigma trata a computação como uma avaliação de funções matemáticas.      Este método enfatiza a aplicação de funções, as quais são tratadas como valores de primeira importância, ou seja, funções podem ser parâmetros ou valores de entrada para outras funções e podem ser os valores de retorno ou saída de uma função. Ex:      -> LISP      -> Haskell      -> Lambda      -> ML(criado em universidade)

Lógico

     Nesse paradigma programas são relações entre Entrada/Saída. Possui estilo declarativo, como o paradigma funcional. Inclui características imperativas, por questões de eficiência. Aplicações em prototipação em geral, sistemas especialistas, bancos de dados, etc. Ex:      -> Popler      -> Conniver      -> QLISP      -> Mercury

Orientado a Obejetos

     A programação Orientada a Objetos é baseada na composição e interação de diversas unidades de softwares denominados objetos. O funcionamento de um software orientado a objetos se dá através do relacionamento e troca de mensagens entre esses objetos. Esses objetos são classes, e nessas classes os comportamentos são chamados de métodos e os estados possíveis da classe são chamados de atributos. Ex:      -> Smalltalk      -> Python      -> Ruby      -> C++

História da Computação

Arquitetura de Von Neumann:

     A Arquitetura de computador de von Neumann se caracteriza pela possibilidade de uma máquina digital armazenar seus programas no mesmo espaço de memória que os dados, podendo assim manipular tais programas. Esta arquitetura é um projeto modelo de um computador digital de programa armazenado que utiliza uma unidade de processamento e uma de armazenamento para comportar, respectivamente, instruções e dados.      A proposta inicial de um computador de programa armazenado, relatada pelo próprio Von Neumann, era dividida em cinco partes:      -> central aritmética;      -> central de controle;      -> memória;      -> gravação;      -> entrada e saída.

Máquina de Turing::

     A Máquina de Turing é um modelo teórico de computação que foi proposto por Alan Turing em 1936. Considerada a base matemática para todas as formas modernas de computação e é uma das contribuições mais significativas de Turing para a ciência da computação.      A máquina é composta de uma fita, dividida em células, cada uma delas pode conter um símbolo da entrada. A fita se move por uma cabeça de leitura/escrita, que pode ler o símbolo atual na célula da fita e escrever um novo símbolo nessa célula.

Linguagem de Máquina:

     A Linguagem de Máquina é um tipo de linguagem de computador que é utilizada para controlar diretamente o comportamento do hardware informático. É utilizada para controlar diretamente o comportamento do hardware de computador, como processadores e memória.      Alguns exemplos são:      -> Assembly      -> Cobol      -> Ada      -> Prolog      -> C      -> C++      -> Java      -> PHP      -> Javascript

Material de ajuda

Propósito/Domínio de Programação

     As linguagens de programação desempenham um papel vital na criação de software, mas não são todas iguais. Cada linguagem é projetada com um propósito específico em mente, e é por isso que os desenvolvedores escolhem diferentes linguagens para diferentes domínios de programação. Os domínios de programação representam áreas de aplicação nas quais as linguagens são mais adequadas e eficazes

Script

     O domínio de programação de script envolve o uso de linguagens de script para automação de tarefas, desenvolvimento web e outras aplicações que requerem facilidade de uso e flexibilidade. Linguagens como JavaScript, Python e Ruby são amplamente utilizadas nesse domínio.      Elas são conhecidas por sua sintaxe amigável e são ideais para criar pequenos programas que automatizam tarefas rotineiras, como processamento de texto, manipulação de arquivos e interação com a interface do usuário.      Além disso, as linguagens de script desempenham um papel importante no desenvolvimento web, onde são usadas tanto no lado do cliente (JavaScript) quanto no lado do servidor (Python, Ruby on Rails). Essas linguagens são ideais para a rápida prototipagem e desenvolvimento ágil de software.

Científico

     O domínio de programação científico abrange o uso de linguagens de programação para modelagem, simulação e análise de dados em campos como física, química, biologia, engenharia e outras disciplinas científicas.      Linguagens como Python, R e MATLAB são populares neste domínio devido à riqueza de bibliotecas e ferramentas disponíveis para análise de dados, visualização e cálculos numéricos.      Essas linguagens permitem que cientistas e pesquisadores desenvolvam modelos complexos e realizem experimentos virtuais, tornando-as essenciais para avanços em pesquisa e inovação científica.

Comercial

     O domínio de programação comercial se concentra no desenvolvimento de aplicativos empresariais, sistemas de gerenciamento, software de contabilidade, recursos humanos, e outras soluções de negócios.      Linguagens como Java, C#, e SQL são amplamente usadas neste domínio devido à sua confiabilidade, desempenho e recursos de integração com sistemas corporativos.      Além disso, frameworks de desenvolvimento, como o Java Spring e o .NET Framework, são comuns para acelerar o desenvolvimento de aplicativos comerciais.      A segurança e a escalabilidade são fatores críticos em projetos comerciais, e essas linguagens oferecem suporte sólido para atender a essas necessidades.

I.A.

     O domínio de programação de Inteligência Artificial (I.A.) é um campo em crescimento que envolve o desenvolvimento de sistemas de software capazes de realizar tarefas que normalmente requerem inteligência humana, como aprendizado de máquina, processamento de linguagem natural e visão computacional.      Linguagens como Python e R são populares neste domínio devido à disponibilidade de bibliotecas de aprendizado de máquina, como TensorFlow, scikit-learn e PyTorch, que simplificam o desenvolvimento de modelos de I.A.      Além disso, linguagens funcionais como Lisp também têm uma longa história em pesquisa de I.A. devido à sua flexibilidade e capacidade de representar conhecimento de maneira eficaz.

Critérios de Avaliação

     A escolha da linguagem de programação é um passo crítico no desenvolvimento de software. Diferentes linguagens atendem a diferentes necessidades e preferências, e a seleção adequada pode afetar diretamente a eficiência e a qualidade do projeto. Para avaliar e selecionar a linguagem de programação mais apropriada, é importante considerar diversos critérios:

Legibilidade

     A legibilidade é um dos aspectos mais críticos na avaliação de uma linguagem de programação. Uma linguagem legível é aquela em que o código-fonte é claro, compreensível e fácil de seguir. Isso não apenas torna a manutenção e a colaboração mais fáceis, mas também reduz a probabilidade de erros. Vários critérios influenciam a legibilidade em uma linguagem de programação: 1. Simplicidade Global:      Uma linguagem deve priorizar a simplicidade global. Isso significa que a linguagem deve ter uma sintaxe geral simples e consistente, evitando construções complexas e ambíguas. Uma sintaxe simples facilita a leitura do código e reduz a probabilidade de erros. 2. Ortogonalidade:      A ortogonalidade em uma linguagem de programação se refere ao grau em que as características da linguagem são independentes e não interferem umas nas outras. Quanto maior a ortogonalidade, mais fácil é aprender e usar a linguagem, pois as regras são consistentes e previsíveis. 3. Instruções de Controle:      As instruções de controle, como condicionais (if, else) e loops (for, while), devem ser intuitivas e fáceis de entender. A clareza das instruções de controle contribui significativamente para a legibilidade do código. 4. Tipos de Dados:      Uma linguagem deve fornecer tipos de dados bem definidos e semântica clara para trabalhar com dados. Isso inclui tipos numéricos, strings, arrays e estruturas de dados. A consistência na representação e manipulação de dados ajuda a melhorar a legibilidade. 5. Estruturas e Sintaxe:      A organização e a estrutura do código, juntamente com a sintaxe da linguagem, desempenham um papel fundamental na legibilidade. Uma linguagem deve permitir uma formatação clara e concisa do código, com identação adequada e uso de espaços em branco significativos. A sintaxe deve ser consistente e seguir convenções amplamente aceitas.

Capacidade de Escrita

     A capacidade de escrita, também conhecida como "writability," é um dos aspectos cruciais na avaliação de uma linguagem de programação. Refere-se à facilidade com a qual um programador pode escrever código de maneira eficiente, concisa e expressiva em uma linguagem. Vários critérios influenciam a capacidade de escrita em uma linguagem de programação: 1. Simplicidade e Ortogonalidade:      Uma linguagem deve priorizar a simplicidade e ortogonalidade. A simplicidade envolve a minimização de construções desnecessárias e complexas, tornando o código mais fácil de escrever. A ortogonalidade refere-se ao grau em que as características da linguagem são independentes e não interferem umas nas outras, o que torna as regras da linguagem previsíveis e coerentes. 2. Suporte para Abstração:      A capacidade de criar abstrações é fundamental. Uma linguagem de programação deve permitir que os desenvolvedores criem abstrações de alto nível para representar conceitos complexos, simplificando a escrita de código. Isso inclui a capacidade de definir funções, classes, módulos e estruturas de dados personalizadas. 3. Expressividade:      A expressividade se refere à capacidade da linguagem de expressar ideias de maneira clara e concisa. Linguagens expressivas permitem que os desenvolvedores comuniquem suas intenções de forma direta, evitando a necessidade de escrever código excessivamente verboso. Isso torna o código mais legível e facilita a escrita.

Simplicidade e Ortogonalidade

     Simplicidade e ortogonalidade são critérios cruciais na avaliação de linguagens de programação. Esses conceitos têm um impacto significativo na legibilidade, capacidade de escrita e manutenção do código. Vamos examinar esses critérios em detalhes: Simplicidade:      A simplicidade em uma linguagem de programação se refere à ausência de complexidade desnecessária. Uma linguagem simples é fácil de aprender e usar, tornando-a acessível a um público amplo. A simplicidade é alcançada através da eliminação de regras e construções desnecessárias, bem como da adoção de uma sintaxe clara e consistente. Linguagens simples são mais propensas a produzir código legível e compreensível. Ortogonalidade:      A ortogonalidade é um conceito que se refere ao grau em que as características da linguagem são independentes e não interferem umas nas outras. Em uma linguagem ortogonal, as regras de combinação de suas características são consistentes e previsíveis. Isso significa que os recursos da linguagem podem ser combinados de várias maneiras sem restrições arbitrárias, o que simplifica a linguagem e a torna mais coerente. A ortogonalidade ajuda a evitar exceções e comportamentos inesperados.      A simplicidade e a ortogonalidade são fatores essenciais para criar uma linguagem de programação eficaz e amigável ao desenvolvedor. A simplicidade torna a linguagem mais fácil de aprender, reduzindo a curva de aprendizado para novos programadores. Ela também torna o código mais legível, facilitando a manutenção e a depuração.

Suporte para Abstração

     A abstração desempenha um papel crucial no design de linguagens de programação, afetando diretamente a facilidade de escrita, que está relacionada à naturalidade da expressão e ao grau de abstração permitido por uma linguagem. As linguagens de programação podem oferecer suporte a duas categorias principais de abstrações: processos e dados.      Para ilustrar a abstração de dados, considere o exemplo de uma árvore binária que armazena dados inteiros em seus nós. Em uma linguagem que não oferece suporte a ponteiros e gerenciamento de memória dinâmica, como Fortran, essa árvore poderia ser implementada utilizando três vetores inteiros paralelos, sendo dois deles usados como índices para especificar nós filhos.      Por outro lado, em linguagens como C++ e Java, é possível implementar essas árvores de maneira muito mais natural, utilizando uma abstração de um nó de árvore na forma de uma simples classe com dois ponteiros (ou referências) e um inteiro. A representação mais natural torna a escrita de programas que envolvem árvores binárias consideravelmente mais fácil nas linguagens C++ e Java em comparação com Fortran.

Expressividade

     A expressividade em linguagens de programação está relacionada à habilidade de realizar tarefas complexas de maneira clara e concisa. Isso é possível graças ao uso de operadores e estruturas de linguagem que simplificam a representação de operações sofisticadas em um código mais legível.      Em Python, por exemplo, a expressividade é evidenciada pelo operador de atribuição (=), que permite realizar atribuições múltiplas em uma única linha. Essa capacidade de expressar operações complexas de forma eficiente é uma característica valiosa em linguagens de programação.

Confiabilidade

     A confiabilidade de um software é alcançada quando ele se comporta de acordo com suas especificações em todas as condições. Isso requer a realização de uma bateria de testes antes que o software seja disponibilizado aos usuários.      Esses testes são essenciais para garantir que o software atenda às expectativas e funcione sem falhas, independentemente das circunstâncias. A confiabilidade é um dos pilares da qualidade de software, garantindo que os usuários possam contar com o programa em todas as situações.

Custo

     O custo no desenvolvimento de software é um fator crítico a ser considerado. Envolve uma alocação de recursos financeiros, tempo e esforço para a criação, manutenção e atualização do software. Esses custos podem variar amplamente dependendo do tamanho e da complexidade do projeto.      A gestão eficaz de custos é essencial para evitar desperdícios e garantir que o desenvolvimento seja economicamente viável. Isso inclui o monitoramento dos recursos, a estimativa precisa de custos e a manutenção de um orçamento adequado ao longo do ciclo de vida do projeto.      A consideração cuidadosa dos custos é crucial para o sucesso de qualquer iniciativa de desenvolvimento de software.
